NSE8 - Start of a journey

2020-07-10 — 12 min read

Since I passed the NSE7 Enterprise Firewall this June, I had to fix myself a new objective for the incoming year. Given that I work mainly with Fortinet products, the choice of NSE8 was pretty obvious. As of today I know I don’t have the knowledge and skills needed to obtain this certification so some planning must be done to close the gap!

First thing to notice, there is only few documentation or feedback from people who passed this certification, even on the Fortinet Reddit’s page … The only reliable information can be found in the handbook from the NSE Institute.

As every respected expert certification level, two exams need to be completed:

  • Written exam: 60 questions (QCM), 120min;
  • Practical exam: 2 days hands-on lab (9h on day 1 and 6h on day 2);

Since I’m working as an network & security engineer, most of the configuration and troubleshooting don’t scare me. I’m exposed to various network topologies, versions, products and of course problems that need to be troubleshooted. I’m more concerned about the corner cases and some deep dive knowledge. I also need to discover some products like FortiADC, FortiWeb, FortiSIEM, etc. that are covered in the certification.

Thanks to my current employer, I have access to the NSE Institute and therefore can access to most of the online formation which covers all the needs for certifications from NSE1 to NSE7. This will help me to understand most of the subjects, at a high level most of the time, and I will have to deep dive using the documentation from CLI references, cookbooks, Fortinet KB, etc. But I’m pretty sure external documentation will be used (FortiGuru, Fortinet subreddit, etc.).

For the first step, the handbook gives us the list of topics that may be tackled during the written lab. We will try to analyze them in order to understand what do we need to learn.

    1. Security Architectures
    • a) Fortinet cloud security solutions : Most of the topics are discussed in the NSE3 & NSE7 Cloud formation;
      • i) FortiGate VM models and licensing : The NSE3 FortiGate Cloud and NSE3 Public Cloud should help on this point;
      • ii) FortiGate VM deployments : Multiple guides are available to deploy VM and work with them on VMware, KVM or even Hyper-v. Some specificity must be studied;
      • iii) Fortinet on private clouds : This point is covered on the NSE7 Private Cloud formation. Two topics will be tackled : NSX-T and OpenStack integration;
      • iv) Fortinet on public clouds : NSE7 Public Cloud formation must cover most of our needs here. AWS, Azure and GCP should be concerned;
    • b) Designing resilient solutions : Resiliency is of course a big subject. This subject is covered in NSE4 and NSE7 formation but a deep dive must be done using cookbooks and KB;
      • i) Fortinet solutions advanced high availability features : Session synchronization, configuration synchronization and failover should be explored here;
      • ii) Fortinet solutions operation and deployment modes
      • iii) FortiGate Cluster protocol : Deep dive in the FGCP protocol, A-P and A-A cluster should have no secret;
      • iv) FortiGate Session Life Support protocol : Deep dive in the FGSP protocol;
      • v) Session-aware load balancing clustering : Discovering and understanding the SLBC. Documentation may be hard to find and labs will not be possible since this deployment concerns chassis;
      • vi) Load balancing : This point must cover active-active cluster load balancing and external load balancing (FortiADC ?);
      • vii) Fault tolerance : I’m not sure what is exactly expected from this point since most of the subject should be covered by the previous topics;
    • c) Designing enhanced Fortinet solutions
      • i) Fortinet solutions for specific scenarios
        • (1) Secure access networks : FortiWiFi and FortiSwitch are covered here. Multiple formation are available from NSE6 WLC/Integrated and Cloud Wireless to NSE7 Secure Access. I may include FortiNAC in this topic;
        • (2) Web application security : Deep dive in the WAF profiles of the FortiGate. The FortiWeb will be covered later;
        • (3) Advanced threat protection : This is more a general topic than a technical one. This subject is covered in the NSE7 ATP formation;
        • (4) Cloud security : Another nebulous topic. FortiGate VM on cloud and other private/public cloud are already covered. Some other products (FortiWeb/FortiMail, etc.) are also covered later. Some newer products like FortiCASB and FortiCWP may be concerned here;
        • (5) Enterprise firewall : The topics tackled in the NSE7 Enterprise Firewall formation are all covered in more specific point;
    • d) Fortinet high-end solutions : We need to understand how these chassis work and the lack of public documentation will not help. Moreover, I will not be able to get my hands on these models since none of our clients have the needs for this kind of products;
      • i) FortiGate 3000 Series
      • ii) FortiGate 5000 Series
      • iii) FortiGate 7000 Series : This topic is covered by a formation on the NSE Institute;
    1. Hardware acceleration : One of the major asset of Fortinet products is the chips they use to handle most of the traffic and security inspection. It is mandatory to understand well how they work;
    • a) Hardware acceleration overview
      • i) Content processors : Content Processors are responsible for processing some CPU intensive security related process;
      • ii) Security processors : Security Processors can be used to accelerate some security specialized process;
      • iii) Network processors : Network Processors work mainly to offload network processing tasks from the CPU;
      • iv) FortiGate blades : This topic should cover how FortiGate blades share resources and how Hardware acceleration is available between multiple blades;
      • v) FortiGate chassis
    • b) Traffic in Fortinet products : The following topics cover the way Fortinet products handle packets, how they offload them to CP, SP or NP, how the Parallel Path Processing works, etc.;
      • i) Traffic offloading
      • ii) Life of a packet
      • ii) Fortinet communication ports and protocols : This topics should allow us to learn what are the port and protocols used by the Fortinet products to works with other Fortinet products or with 3rd party products;
    1. Networking
    • a) Advanced routing : Routing is of course one of the main functionality of FortiGate and should be studied well;
      • i) Static routing
      • ii) Dynamic routing : e/iBGP, OSPF, IS-IS & RIP usage on the FortiGate must be studied;
      • iii) Routing and high availability : Concepts regarding routing operation on HA must be explained, such as how standby unit handle routing table, how BGP peering or OSPF adjacencies are maintained, etc.;
      • iv) Asymmetric routing : In some case, it can be needed to allow FortiGate to do asymmetric routing and we have to understand the underlying concepts;
      • v) Secure SD-WAN : FortiGate integrates some SD-WAN capabilities. I’ll explore what we can do with this either for remote branches interconnection or for a dynamic Internet access;
    • b) Advanced NAT : Since FortiGate mostly act as a L3/L4 equipment, they can NAT traffic;
      • i) Central NAT : It is possible to configure NAT using firewall policies or by creating NAT policies. I’ll try to cover differences between these modes;
      • ii) NAT64 : This point should cover how FortiGate handle NAT from IPv6 to IPv4. This is used with DNS64;
      • iii) NAT46 : FortiGate can also handle NATing traffic from IPv4 to IPv4;
      • iv) DNS64: Used with NAT64, it allows IPv6-only capable clients to access IPv4-only services using DNS modification;
    • c) VPN technologies
      • i) Advanced IPsec : This topic must cover some advanced IPsec concept such as ADVPN, OCVPN, aggregate/redundant, etc.;
      • ii) Advanced SSL : FortiGate allows administrators to configure things like SSL-VPN realm, RADIUS authentication, MFA/2FA, etc.;
    • d) Network troubleshooting
      • i) Debugging tools : FortiOS includes debugging tools used to capture/sniff traffic which we will try to understand;
      • ii) Diagnostic tools
      • iii) Routing and VPN troubleshooting : We need to understand how to troubleshoot problems concerning routing (static, dynamic, PBR or even SD-WAN) and VPN (IPsec or SSL);
      • iv) CLI in Fortinet solutions
    1. Authentication
    • a) Authentication with Fortinet products
      • i) Advanced single sign-on : FortiGate support some SSO capabilities that allows to filter traffic based on users/groups instead of IP addresses;
      • ii) RADIUS : It is possible to use RADIUS as authentication source for multiple purposes (SSL-VPN, Administrators, RSSO, etc.);
      • iii) Two-factor authentication : It is possible to support 2FA/MFA with FortiTokens, hardware or software, directly on the FortiGate;
      • iv) 802.1x : When FortiGate manage FortiSwitch it is possible to configure 802.1x profiles and apply them on the switches ports;
      • v) Certificate-based authentication : Even if username/password authentication is mainly used, it is also possible to use certificates to authenticate users and administrators;
      • vi) Troubleshooting
    1. Content inspection : Of course, as a NGFW, FortiGate can inspect traffic so we have to understand what traffic can be inspected and how it works;
    • a) Inspection modes
      • i) SSL/SSH : It is possible to perform a MitM and analyze SSL/TLS or SSH traffic. We have to understand how this works, what are the limitation and how to deploy it;
      • ii) Certificate : When TLS interception is not possible, it is possible to just analyze certificates;
      • iii) Sandbox inspection : For some protocols it is possible to send intercepted files to a Sandbox to analyze them;
    • b) FortiOS security profiles
      • i) Antivirus : Files and documents intercepted on supported protocols can be analyzed. We have to understand what can be analyzed & how works the AV engine;
      • ii) Intrusion prevention systems (IPS) : Traffic can be analyzed to detect known malicious signatures;
      • iii) Application control : Analyzing L7, FortiGate can apply fine grained policies to filter applications either web based or not;
      • iv) Web filtering : In order to allow user to access web resources but with some security enforcements, this functionality will filter websites based on their reputation or their category;
      • v) DNS filtering : Similar to the web filtering, this will take place at the DNS level, before any HTTP/S request;
      • vi) VoIP inspection : FortiGate can analyze SIP sessions and apply action based on some criteria;
      • vii) Session helpers : FortiGate can also analyze other protocols in order to provide some help establishing connection (FTP data channel is a good example);
    • c) FortiGuard services
    1. Security operations
    • a) Fortinet solutions APIs
      • i) Rest API : Some, if not most, Fortinet products come with a REST API we will try to explore and use some tools to help us is this task;
    • b) Handling security events with Fortinet solutions
      • i) FortiSIEM : As stated in its name, FortiSIEM is a Security Information & Event Management. We will try to understand how it works and how we can integrate it in the Fortinet ecosystem and with 3rd party devices;
      • ii) FortiAnalyzer : Preconized solution to collect logs from Fortinet products, it has an easy integration and provides key in hand dashboard, reports and alerting;
      • iii) Log analysis : FortiGate and other Fortinet product generate many logs and it is important to understand what they mean and where we can find interesting information;
      • iv) Event management
    • c) Fortinet central management solutions
      • i) FortiManager : When many FortiGate need to be managed, FortiManager will help to configure them, provision them, update them, etc.;
      • ii) FortiCloud : FortiCloud consist of multiple online (SaaS) tools that helps network administrator to manage their Fortinet’s products. FortiCloud also provides some free tools for security or log management;
    1. Integrated Solutions : Fortinet offer the possibility to integrate a full ecosystem of products that can interact. It is the case for the FortiAP (WiFi Access Point) and FortiSwitch (Switches) but other products also help for security purposes;
    • a) Integrate Fortinet solutions for advanced threat protection : When security is a biggest concern, it is possible to include external product, either On-Premise or SaaS, to prevent, detect or mitigate threat. FortiSandbox is the main example but other products can be interconnected with the Fortinet’s Security Fabric;
    • b) FortiLink : Main component of the FortiSwitch architecture, FortiLink is used to manage FortiSwitches and serves as an uplink for the FortiSwitches;
    • c) Fortinet wireless solutions : Fortinet provides the ability to deploy either FortiAP or FortiWifi. The first ones are classic WiFi Access Point that are controlled by a FortiGate. The second ones are classic FortiGate with an internal WiFi Access Point;
    • d) Fortinet authentication solutions : FortiAuthenticator & FortiToken, either Cloud or not, are solutions available to manage identity and authentication for the Fortinet products and, of course, for 3rd party products;
    • e) Fortinet hybrid solutions (traditional network and cloud)
    1. Enhanced Technologies
    • a) Fortinet enhanced solutions including
      • i) FortiWeb : Web Application Firewall solution, Firewall helps to protect web applications and API against threat actors;
      • ii) FortiADC : As an Application Delivery Controller, FortiADC helps to publish application, protect them, make them resilient;
      • iii) FortiDDoS : *FortiDDoS will protect network against Distributed Denial of Service analyzing traffic from layer 3 to layer 7, using behavior analysis, known threat source, etc. *
      • iv) FortiAuthenticator : Identity and authentication source, FortiAuthenticator can aggregate multiple identity source, force MFA/2FA usage;
      • v) FortiMail : Since emails are one of the major compromise source, they need to be analyzed, filtered and sanitized. FortiMail helps to achieve this;

Since I work daily with Fortinet products, I may also cover newer version and maybe compare or comment some changes in the behavior of some functionalities between version.

NSE8 covers a lot of security aspects, that is why I’ll also keep some time to read other security information related papers and listen to my favorites security podcast while driving.

As you can see there are many topics to cover on many protocols, technologies and products. To help me, I’ll try to make some flashcards, mainly on topics I don’t know yet and I’ll try to share them.

I don’t feel I can have a fixed study planning since I move very often at the premises of customers for whom I work. So it can happens that I’m not at home for a week or more and it’s hard to predict how long my days can be. Anyway I’ll try to keep a steady pace most of the time this means :

  • At least one 1h every day to learn something new, either reading documentation, watching a product presentation, reading blogs, etc.;
  • At least 1/2h of revision with my flashcard or writing some part of a blogpost every two day;
  • At least 4h of lab every week-end either Saturday or Sunday depending of my personal planning;

My actual objective is to pass the written exam in 2021, between April and May and the practice lab this summer.
